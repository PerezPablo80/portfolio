    Tendermeter

    El Tendermeter es un desarrollo innovador de medición de terneza de la carne.

    Dicho desarrollo fue realizado en Java, actualmente está hecho con un SBC Raspberry Pi 4b y una pantalla LCD1602 que se conecta con un desarrollo hecho en Python. Inicialmente se hicieron pruebas con Udoo v8, Udoo v3, Asus thinkerboard y hasta un Rpi2, pero por un requerimiento de velocidad y disponibilidad, se terminó optando por RPi4 y potencialmente 5.

    Dejamos un video del prototipo, dado que no se puede presentar el proyecto finalizado aún porque el dispositivo aún no se encuentra en el mercado.

    La proyección es que el mismo estará finalizado a principios de 2025.

    Mis aportes en este proyecto fueron el desarrollo, el diseño y la arquitectura de software, así como de hardware.

    App de Servicio técnico de Jade Informática

    Durante los primeros años de Jade, trabajé en el desarrollo con php (framework fat free framework o F3), mysql, html, CSS, js que es el sistema que actualmente funciona en la empresa para tickets de servicios.

    En los últimos años se ajustaron algunos componentes y se realizó la transición de php 5 a 8

    El mismo cuenta con un sistema de clientes, dispositivos asociados a los mismos, usuarios y los tickets de servicio.

    El servicio se puede validar con el usuario, haciéndole firmar el mismo en cualquier celular con conexión a internet.

    Dicha validación habilita el envío de un mail al cliente, si el campo fue ingresado.

    Así mismo los servicios cargados nos llegan a un mail con el inicio del proceso y posteriormente cuando se firma o finaliza, nos envía la información para proceder a facturar dicho servicio, si corresponde. Los servicios son realizados por los técnicos.

    Demo de Parking

    Un cliente particular solicitó una demo de un sistema de parking. El mismo se realizó utilizando electronJS.

    Dado que era una demo y que no se requiere inicialmente una base de datos, se utilizó un archivo de excel para guardar la información y para mostrar potencialmente el sistema.

    El sistema permite ingresar el numero de Chapa del auto, y lo ingresa al listado de pendientes, guardando el mismo en Excel.

    Al guardar en Excel se imprime en una impresora térmica un ticket con un código de barra. Esto finaliza el primer proceso.

    Luego al salir se selecciona la chapa del auto y se verifica el tiempo, para lo cual se realizan los cálculos correspondientes.

    Cuando el cliente abona se guarda el pago, que puede o no coincidir con el importe final y se emite un ticket de salida.

    El ticket de salida no fue realizado, a diferencia del resto de la demo.

    Notese que el pago del cliente puede ser distinto de lo que indica el ticket, esto se debe a un requerimiento del cliente quien explica que las propinas pueden estar consideradas en el cálculo.

    El cliente entiende que se puede abonar con tarjeta y la propina incluirse en el pago, lo cual se busca cuantificar mensualmente en la aplicación.

    Para acceder al git: git parking printer. El mismo es de dominio público también.
